Comparative analysis of Linear Regression, Support Vector Regression, and K-Nearest Neighbors on the Boston Housing dataset (MEDV target).

Project 1
Linear Regression
Fits a straight-line relationship between input features and the median house value. Simple, interpretable, but assumes linearity.
Project 2
Support Vector Regression
Uses a hyperplane with epsilon-insensitive margins in a higher-dimensional space. Handles non-linearity with kernel tricks.
Project 3
K-Nearest Neighbors
Predicts based on the average of K closest training samples. Non-parametric and flexible, but sensitive to feature scale.

Model Profile Analysis

The radar chart maps each algorithm across four normalized performance dimensions. SVR dominates across all axes, while KNN holds second place. Linear Regression lags significantly, indicating the Boston Housing dataset has non-linear relationships that a simple linear model cannot capture.

Key Takeaway: The feature RM (average rooms) and LSTAT (% lower-status population) are highly correlated with MEDV. Kernel-based methods like SVR capture these non-linear patterns far better than a straight-line fit.
